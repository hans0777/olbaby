## 🔧 배포 방법 (3단계)

### 1단계: GitHub 저장소 생성 (5분)

1. **GitHub 웹사이트 접속**
   - https://github.com 접속
   - 로그인 (계정: hans97490800@gmail.com)

2. **새 저장소 생성**
   - 오른쪽 상단 `+` 버튼 클릭 → "New repository" 선택
   - 또는 직접 이동: https://github.com/new

3. **저장소 설정**
   ```
   Repository name: olbaby
   Description: 오엘베이비 고객관리 자동화 시스템
   Visibility: Public (GitHub Pages는 Public 저장소 필요)
   
   ❌ Initialize this repository with:
      - README 체크 해제
      - .gitignore 체크 해제
      - license 체크 해제
   ```

4. **"Create repository" 버튼 클릭**

---

### 2단계: 로컬 저장소와 GitHub 연결 (2분)

#### PowerShell에서 실행:

```powershell
# 1. 프로젝트 폴더로 이동
cd "c:\Users\hans\OneDrive\바탕 화면\claud cowork\olbaby"

# 2. GitHub 저장소와 연결 (YOUR_USERNAME을 실제 GitHub 사용자명으로 변경)
git remote add origin https://github.com/YOUR_USERNAME/olbaby.git

# 3. 기본 브랜치 이름 확인/변경
git branch -M main

# 4. 첫 푸시 (GitHub에 업로드)
git push -u origin main
```

#### 사용자명 확인 방법:
- GitHub 로그인 후 오른쪽 상단 프로필 클릭
- "Your profile" → URL에서 확인
- 예: `https://github.com/username` → `username` 부분

#### 인증 방법:
- **방법 1: Personal Access Token (권장)**
  - GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic)
  - "Generate new token" → repo 권한 체크 → 생성
  - 토큰 복사하여 비밀번호 대신 입력

- **방법 2: GitHub Desktop 사용**
  - GitHub Desktop 다운로드: https://desktop.github.com/
  - 로그인 후 저장소 추가

---

### 3단계: GitHub Pages 활성화 (2분)

1. **GitHub 저장소 페이지 접속**
   - https://github.com/YOUR_USERNAME/olbaby

2. **Settings 탭 클릭**

3. **왼쪽 메뉴에서 "Pages" 클릭**

4. **Source 설정**
   ```
   Source: Deploy from a branch
   Branch: main
   Folder: / (root)
   ```

5. **"Save" 버튼 클릭**

6. **배포 완료 대기 (약 1-2분)**
   - 페이지 상단에 배포 URL 표시
   - 예: `https://YOUR_USERNAME.github.io/olbaby/`

## 🔧 문제 해결

### 1. git push 실패 (인증 오류)
```
❌ Authentication failed
```

**해결방법:**
- Personal Access Token 재생성
- GitHub Desktop 사용 권장

### 2. GitHub Pages가 404 오류
```
❌ 404 There isn't a GitHub Pages site here.
```

**해결방법:**
- Settings → Pages → Source 재설정
- main 브랜치 선택 확인
- 1-2분 대기 후 새로고침

### 3. 파일이 업데이트되지 않음
```
❌ 최신 변경사항이 반영되지 않음
```

**해결방법:**
- 브라우저 캐시 삭제 (Ctrl + Shift + Delete)
- 시크릿 모드로 접속
- GitHub Pages 배포 완료 확인 (1-2분 소요)
